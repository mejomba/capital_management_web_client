# CLAUDE.md — ریپو فرانت‌اند (Wealth Manager Frontend)

این فایل قواعد قطعی ریپو فرانت است. قبل از هر تغییر بخوان و رعایت کن.
اسناد مرجع کنار همین فایل: `PRD.md`، `GLOSSARY.md`، `API_CONTRACT.md`، و بخش M6 در `ROADMAP.md`.

---

## ۱. نقش این ریپو
فرانت **فقط مصرف‌کننده‌ی API** است. منبع حقیقت داده و منطق مالی در بک‌اند است.
هدف: نمایش دقیق، خوانا و فارسی/RTL از داده‌هایی که بک می‌دهد.

## ۲. قاعده‌ی کلیدی: منطق مالی را در فرانت بازتولید نکن
> سود/زیان، FIFO، ارزش خالص، XIRR/TWR و تبدیل ارز همه در بک‌اند محاسبه می‌شوند.
> فرانت این اعداد را **نمایش** می‌دهد، نه اینکه دوباره حساب کند.
اگر دیدی داری در فرانت P&L یا تبدیل ارز محاسبه می‌کنی، اشتباه است — آن را از API بخواه.
استثنای مجاز: فرمت/نمایش (گرد کردن نمایشی، جداکننده‌ی هزارگان، تبدیل تاریخ به شمسی).

## ۳. استک
- **React 18 + TypeScript + Vite**.
- داده‌ی سرور با **React Query (TanStack Query)** — نه نگه‌داری دستی state سرور.
- نمودار با **Recharts**. کامپوننت‌لایبرری: shadcn/ui یا MUI (یکی را انتخاب و ثابت بمان).
- فرم‌ها با react-hook-form + zod (هم‌خوان با schemaهای API).

## ۴. قرارداد API و تایپ‌ها (مهم‌ترین قاعده‌ی جداسازی)
- تایپ‌ها را **دستی ننویس**. از `openapi.json` بک‌اند با `openapi-typescript` (یا `orval`) **کلاینت تایپ‌دار تولید کن**.
- یک اسکریپت `npm run gen:api` که تایپ‌ها/کلاینت را از OpenAPI می‌سازد؛ خروجی در `/src/api/generated`.
- اگر قراردادی در بک عوض شد، با تولید مجدد تایپ‌ها، خطاهای TypeScript جای ناهماهنگی را نشان می‌دهند.
- `API_CONTRACT.md` این‌جا فقط مرجع خواندنی است؛ منبع حقیقت `openapi.json` است.

## ۵. پول و عدد
- اعداد پولی از API به‌صورت **رشته‌ی Decimal** می‌آیند. برای محاسبه‌ی نمایشی (اگر لازم شد) از کتابخانه‌ی
  دقیق مثل **decimal.js** استفاده کن، نه `Number`/`parseFloat`. هرگز پول را در float جاوااسکریپت نریز.
- فرمت نمایش: جداکننده‌ی هزارگان فارسی، واحد ارز کنار عدد.

## ۶. ارز دوگانه (IRR + USD)
- یک **سوییچ ارز سراسری** (IRR / USD / هر دو) که روی همه‌ی گزارش‌ها و داشبورد اثر بگذارد.
- مقدار هر ارز را از API بخواه (`?currency=`)؛ خودت بین ارزها تبدیل نکن.

## ۷. تاریخ شمسی (Jalali)
- API تاریخ‌ها را ISO/میلادی می‌دهد. **نمایش و انتخابگر تاریخ شمسی** کارِ فرانت است.
- از کتابخانه‌ی شمسی (مثل dayjs-jalali یا jalaali-js) برای تبدیل نمایش و بازه‌های ماهانه/سالانه استفاده کن.
- فیلتر گزارش‌ها بر مبنای ماه/سال شمسی، اما در ارسال به API به ISO تبدیل شود.

## ۸. RTL و زبان
- چیدمان کامل **RTL** و رابط فارسی. فونت فارسی مناسب (وزیرمتن/ایران‌سنس).
- اعداد و نمودارها در حالت RTL درست نمایش داده شوند (محور، tooltip، جهت).

## ۹. احراز هویت
- JWT را امن نگه دار؛ روی هر درخواست هدر `Authorization` بگذار (interceptor در لایه‌ی API).
- مدیریت انقضا/۴۰۱ → هدایت به ورود. مسیرهای محافظت‌شده با guard.

## ۱۰. ساختار و کیفیت
```
/src
  /api          # کلاینت + /generated (از OpenAPI)
  /features     # dashboard, accounts, assets, transactions, prices, liabilities, goals, reports
  /components   # کامپوننت‌های مشترک UI
  /lib          # فرمت عدد/تاریخ شمسی، helperها
  /hooks        # React Query hooks
  /routes
```
- منطق دریافت داده در hookهای React Query؛ کامپوننت‌ها تا حد امکان presentational.
- وضعیت‌های loading/error/empty را برای هر داده‌ی سرور هندل کن.
- خوانایی و دسترس‌پذیری (a11y) را رعایت کن.

## ۱۱. صفحات فاز اول (M6)
داشبورد (ارزش خالص + ترکیب دارایی)، حساب‌ها/دارایی‌ها، ثبت تراکنش (فرم مخصوص هر نوع: deposit/withdrawal/trade/transfer/income)،
قیمت‌ها (ورود دستی)، بدهی‌ها، اهداف (با progress)، گزارش‌ها (بازدهی، تورم، پیش‌بینی).

## ۱۲. روال کار با Claude Code
اول `openapi.json` بک را بگیر و کلاینت تایپ‌دار بساز، بعد صفحه‌به‌صفحه جلو برو.
در هر مرحله **اول پلن و لیست فایل‌ها**، بعد کد. هیچ endpoint را فرض نکن؛ از قرارداد/تایپ‌های تولیدشده استفاده کن.
